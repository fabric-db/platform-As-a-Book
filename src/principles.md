# Principles

## The Book Is Canonical

If the platform has a rule, the book must be able to explain it.

If the runtime has a behavior, the book must be able to name it.

If an agent has authority, the book must define where that authority comes from.

## Governance Before Automation

Automation without governance creates speed without accountability.

The platform must define identity, permissions, policy, approval, audit, and revocation before agents can act with real authority.

## Data Carries Meaning

Data is not only storage. Data carries identity, provenance, classification, access, and context.

The platform treats data as fabric: connected, governed, and traceable across systems.

## Agents Are Accountable Actors

An agent is not just a workflow. It is an accountable actor with identity, scope, permissions, objectives, evaluation, and lifecycle state.

## Runtime Is Reconciliation

The platform is never finished. It continuously compares desired state against actual state and records the difference.

## Humans Stay At The Gate

Critical decisions remain reviewable. The platform should increase human leverage, not hide decisions behind automation.
