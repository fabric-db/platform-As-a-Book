# Protocols

Protocols make the platform interoperable.

The book defines the protocol intent before implementation.

## Core Protocol Families

| Protocol area | Purpose |
| --- | --- |
| Identity | Who is acting and on whose behalf |
| Access | What the actor is allowed to do |
| Context | What information is available |
| Agent | How agents describe capability and lifecycle |
| Tool | How tools are discovered and invoked |
| Event | How runtime changes are emitted |
| Payment | How commitments and settlements are governed |
| Evidence | How actions are audited and verified |

## Protocol Rule

A protocol must define:

- actors
- messages
- permissions
- failure modes
- audit requirements
- versioning
- compatibility rules

## Open Box Protocol

The Open Box Protocol describes how governed units move through Fabric while preserving identity, source, policy, and provenance.
