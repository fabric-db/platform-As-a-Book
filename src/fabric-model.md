# Fabric Model

Fabric is the connected layer where data, identity, access, policy, agreement, and runtime context meet.

Fabric is not only a database. It is a governed graph of meaning.

## Fabric Responsibilities

Fabric must:

- preserve source meaning
- connect data across systems
- enforce identity and access
- carry provenance
- support temporal history
- expose policy context
- provide audit evidence
- enable agent reasoning without exposing more data than necessary

## Fabric Flow

```text
Source
  -> Box
  -> Fabric
  -> Policy
  -> Agent
  -> Decision
  -> Evidence
```

## Fabric As Governance Surface

Fabric is where governance becomes queryable.

Instead of asking only "where is the data?", the platform can ask:

- Who owns this?
- Who can use this?
- What policy applies?
- What identity is acting?
- What agreement allows this?
- What evidence proves it?
